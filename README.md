       o  Некоторые стили CSS

•	Использовать паттерны проектирования(Factory, Builder).

Был реализован следующий функционал:
•	Запуск приложения с консоли (Инструкция внизу).
•	Работа с файлами:
       o  Набор исходных параметров находится в файле data.txt (путь к файлу C:\check\build\resources\main\ data.txt)
       o  Вывод сформированного чека на основании входных параметров 
          в файле check.txt (путь к файлу C:\check\build\resources\main\ check.txt)
•	Предусмотрены акционные товары. Если их чеке больше пяти, то делается скидка 10% по этой позиции. Информация отображается в чеке.
•	Продемонстрирована работа с БД. Набор товаров (добавлено 27 позиций) и скидочных карт (добавлена 1 позиция) хранится в таблицах БД.
•	Реализована обработка иключений (товара с id или файла не найдено и др.)
•	Покрыта часть функционала юнит-тестами.
•	Реализован REST- интерфейс( пердусмотрена возмодность получения чека по GET.
        Например: http://localhost:8081/check?itemId=1&itemId=2, где itemId- id продукта в покупке). Реализован на Spring Boot.

Инструкция по запуску:

Небходимо скачать файл в корень диска C:\. На компьютере должна быть установлена БД  PostgreSQL 15.1-1. Необходимо импортировать файл БД: check_db.sql (путь к файлу c:\check\src\main\resources\ check_db.sql). Запустить командную строку в директории C:\check\build\libs> и выполнить команду:  java -jar "C:\check\build\libs\check-0.0.1-SNAPSHOT.jar"
